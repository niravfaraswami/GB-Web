# GutBasket — Blog Post Template (Template 09)

**Source:** `gutbasket-blog.html`
**Maps to in Shopify:** `templates/article.json` (or a `.liquid` template) + a custom section `sections/article-main.liquid`. Use this for the **Learn** content type — long-form editorial articles by category (Fermentation 101, Gut Health Science, Troubleshooting).

---

## What this template is for

Long-form editorial articles. ~1,500–3,000 words. Mixed prose with the occasional inline product callout, pull quote, and a sources list at the bottom. Sticky table of contents on the side. Hard "buy the kit" CTA at the end. Two related-article cards.

Think Substack-style readability, not Medium — narrower text column, generous leading, real typographic care.

---

## File layout

The HTML file is a fully renderable preview. Three logical zones inside `<body>`:

1. **Theme shell** — promo bar, `<header class="site-header">`, and `<footer class="site-footer">`.
   *Already exists in the live theme. Ignore when porting — these become Shopify sections (`sections/header.liquid`, `sections/footer.liquid`) that wrap every page.*

2. **Template body** — everything inside `<main>`. This is what becomes the Shopify article template / section.

3. **Inline `<script>`** — TOC scroll-spy and smooth-scroll. Port as-is into the section's `{% schema %}`-less script tag, or move to `assets/article.js`.

## Sections inside `<main>` (in order)

| # | Section | CSS root class | Required content |
|---|---|---|---|
| 1 | Article hero | `.article-hero` | Category pill, H1 title, italic subtitle, byline (author · role · date · read time), feature image/emoji block |
| 2 | Article body wrap (TOC + prose) | `.article-body-wrap` | Sticky TOC sidebar (`<aside class="article-toc">`), `<article class="article-body">` containing N `<section class="article-section" id="...">` blocks |
| 2a | Inline pull-quote (optional, inside body) | `.pull-quote` | Single sentence in italic, kit-color left border |

| 2c | Sources & further reading (end of body) | `.sources-block` | Numbered ordered list, italic book/paper titles |
| 3 | End-of-article CTA | `.end-cta-section` > `.end-cta-card` | Hard product push: dark card, kit-color accent, eyebrow + title + desc + product name + price + button |
| 4 | Related articles | `.related-section` | 2-card grid, one card per related article, category dot in card color, title, read time |

## Color / category theming

The template uses three CSS variables for **all** accents:

```css
--kit-color:       /* solid accent — pills, links, buttons, progress bar */
--kit-color-dark:  /* hover state, deep accent */
--kit-color-soft:  /* tinted background — pill bg, quote bg */
```

These are set per article based on **category**. Three categories exist in the system:

| Category | `--kit-color` | `--kit-color-dark` | `--kit-color-soft` |
|---|---|---|---|
| Fermentation 101 | `#FF7300` (achaar orange) | `#E66800` | `#FFE4D1` |
| Gut Health Science | `#15BA97` (teal) | `#0F9279` | `#D5F2EA` |
| Troubleshooting | `#C24528` (brick red) | `#9A361F` | `#F5D9D0` |

In Shopify, set these on `<body>` inline based on `article.metafields.category`:

```liquid
<style>
  :root {
    --kit-color:      {{ article.metafields.custom.color | default: '#FF7300' }};
    --kit-color-dark: {{ article.metafields.custom.color_dark | default: '#E66800' }};
    --kit-color-soft: {{ article.metafields.custom.color_soft | default: '#FFE4D1' }};
  }
</style>
```

The current preview file is themed for **Fermentation 101 (orange)**.

## Data model — what to wire from Shopify

| HTML hook | Shopify source |
|---|---|
| `[data-bind="category"]` text | `article.metafields.custom.category_label` (e.g. "FERMENTATION 101") |
| `<h1 data-bind="title">` | `article.title` |
| `.article-subtitle [data-bind="subtitle"]` | `article.excerpt` (richtext stripped) or `article.metafields.custom.subtitle` |
| `[data-bind="author"]` | `article.author` |
| `[data-bind="author-role"]` | `article.metafields.custom.author_role` (default "Brand Editorial") |
| `[data-bind="date"]` | `article.published_at \| date: "%B %d, %Y"` |
| `[data-bind="read-time"]` | `article.metafields.custom.read_time` or computed from word count (~225 wpm) |
| `[data-bind="hero-emoji"]` (or replace with image) | `article.image` if set, else `article.metafields.custom.hero_emoji` |
| TOC list (`#toc-list`) | Generated from `article.metafields.custom.toc` JSON `[{id, label}, ...]` OR auto-generated by parsing `<h2>` ids in body |
| `<section class="article-section" id="...">` blocks | `article.content` rendered as rich text. Recommend a custom blocks-based section schema for TOC anchors. |
| `.inline-callout` blocks | Authored as a custom block type (or as `<custom-callout product-handle="...">` shortcodes parsed in the article body) |
| `.pull-quote` | Authored as a `<blockquote class="pull-quote">` in rich-text (preserve class through richtext filter) |
| `.sources-block ol li` | `article.metafields.custom.sources` rich-text or list metafield |
| End CTA fields (eyebrow, title, desc, product, price) | Single product reference metafield + override metafields |
| Related articles | `article.metafields.custom.related[2]` references, or auto-pull from same blog with same `tag` |

**Recommendation:** use Shopify's article `content` for prose, plus a **section schema** with these blocks:
- `block: hero` (category, title, subtitle, byline, image)
- `block: section` (heading + content; produces `<section id>` with H2)
- `block: pull_quote`
- `block: inline_callout` (product reference + override copy)
- `block: sources` (list)
- `block: end_cta` (product reference + override copy)
- `block: related_articles` (list of article references)

This gives editors a clean experience and avoids fighting with Shopify's rich-text editor.

## Typography rules used

- **H1** — Montserrat 800, 48px, `letter-spacing: -0.025em`, line-height 1.1, no uppercase.
- **H2 (in body)** — Montserrat 800, 28px, `letter-spacing: -0.015em`, line-height 1.2.
- **Body prose** — DM Sans 17px, line-height 1.7. (Note: prose is **17px**, not the typical 16px — this is intentional for long-form readability.)
- **Eyebrows / TOC labels / sources / pill** — DM Mono, 10–11px, `letter-spacing: 0.12–0.14em`, uppercase.
- **Pull quote** — Montserrat 600, 20px, italic, kit-color left border.
- **Subtitle** — DM Sans italic 19px.

Keep this — do not normalise to a generic 16px body or system font stack.

---

## Layout / breakpoints

- Article hero column max-width: **760px** (narrow — reading column).
- Body wrap container max-width: **1100px**, two-column grid `240px 1fr` (sidebar + body), gap 56px.
- `.article-body` capped at **720px** — actual reading column inside the 1fr cell.
- Below 768px: TOC collapses to a horizontal block above the article (no longer sticky), grid goes single-column. All sections in the file's `@media (max-width: 768px)` block apply.

---

## JS to preserve

Two behaviours, both in the inline `<script>` at the bottom of the file:

1. **TOC smooth-scroll** — clicking a TOC anchor scrolls smoothly to the section.
2. **Scroll-spy** — as the reader scrolls, the active TOC link is highlighted. Logic: find the section whose `offsetTop - 120` is closest to but below `window.scrollY`.

Port both. They're plain DOM API, no dependencies.

---

## Things to NOT do when porting

- **Don't** rip out the kit-color CSS variables and hard-code colors. The whole theming system depends on these three vars.
- **Don't** convert the Montserrat/DM Sans/DM Mono triad to system fonts. The brand identity is in the typography.
- **Don't** widen `.article-body` past 720px. The 720px cap is what makes long articles actually readable.
- **Don't** add stock-image blog hero patterns. The current design uses a solid `--kit-color-soft` background block with a large emoji (or nothing) on purpose — feels editorial, not corporate.
- **Don't** turn the inline product callout into a full-card. Its restraint is the point — it sits inside prose without breaking the reading flow.
